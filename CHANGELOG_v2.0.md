## 项目概述

智心树洞是面向温州科技职业学院数智技术学院的微信小程序，旨在为学生与辅导员/教师提供受保护的沟通渠道，降低求助门槛，并帮助教师更早识别和处理学生心理风险。

**核心目标：**
- 保护学生隐私，降低表达与求助门槛。
- 为教师提供及时预警与跟进渠道。
- 为后续 AI 辅助能力预留扩展空间。

**目标用户：**
- 学生（在校大专生）
- 教师 / 辅导员（负责心理关怀与管理）

**接入平台：** 微信小程序，嵌入学院公众号。

## 核心功能

### V1.0 基础能力
- 问题广场：展示公开问题与回答，师生均可参与，支持实时内容审核。
- 匿名树洞：支持向指定教师或树洞匿名/实名留言，可选择是否公开到广场。
- 教师端：查看留言、私密回复、处理公开回答、接收通知、导出对话记录。
- 权限体系：区分学生、教师、管理员，支持分级访问控制与审计。

### V2.0 强化能力
- 教师邀请码升级：通过 `TEACHER_INVITE_CODE` 完成教师身份自助升级。
- 教师资料管理：教师可维护公开资料与在线状态。
- 广场管理后台：支持广场内容审核、管理与删除。
- 审核与通知优化：审核逻辑进一步后移到服务端，前端提示更轻量。

## 功能流程概述

1. 学生在首页进入问题广场或树洞入口。
2. 选择目标教师，填写内容，并设置匿名/公开选项。
3. 提交后触发内容审核，通过后进入公开广场或私密对话。
4. 教师在工作台处理留言、回复学生或管理公开问题。
5. 遇到高风险事件时，按审批流程上报并执行紧急干预。

## 技术架构

- 前端：微信小程序（WXML / WXSS / JavaScript）
- 后端：Python Flask + SQLAlchemy，Gunicorn（WSGI）
- 异步：Celery + Redis（通知、异步任务）
- 数据：MySQL 8.0（主库），Redis（缓存与队列）
- 部署：Docker + Docker Compose，云服务器（阿里云 / 腾讯云）
- 集成：微信内容安全 API、钉钉 Webhook、微信公众号入口

**技术取舍：**
- Flask 适合快速迭代 MVP。
- Python 生态便于后续扩展 AI 功能。
- MySQL 适合结构化的用户、留言和权限数据。

## UI / UX 设计要点

- 明显区分公开与私密场景的视觉系统：广场偏灰，树洞偏绿。
- 发布前提供公开/匿名确认，降低误操作风险。
- 头像加载异常时自动降级为文字头像，保证页面稳定。
- 对话页强调清晰消息流与即时发送反馈。

## 数据安全与合规

- 敏感数据加密存储，访问按角色与审批控制并记录审计日志。
- 数据默认保留 30 天，可按制度调整。
- 紧急身份揭示需明确触发条件、审批流程与操作留痕。
- 敏感凭据通过环境变量或密钥管理服务注入，不在代码中硬编码。

## 风险与阻塞

### 伦理与法律风险
- **假匿名问题**：学生以为匿名，但教师能看到真实身份。
  - 风险：学生信任度下降，可能引发投诉。
  - 缓解：仅在紧急情况（如自伤风险）下查看身份，并建立明确使用规范。
- **数据隐私合规**：学生心理健康数据属于敏感信息。
  - 风险：违反数据保护法规。
  - 缓解：咨询法务部门，确保符合《个人信息保护法》。

### 技术风险
- **内容审核误判**：AI 审核可能误判正常内容或漏判敏感内容。
  - 缓解：建立人工复审机制，并维护敏感词库。
- **极端情况响应**：学生表达自伤倾向时的紧急处理。
  - 缓解：建立紧急通知机制（钉钉 + 短信），制定应急预案。

### 运营风险
- **使用率低**：学生不信任或不使用平台。
  - 缓解：加强初期宣传，清晰说明隐私保护机制。
- **问题广场冷清**：缺少提问或回答。
  - 缓解：由教师主动发起话题，并考虑积分/徽章等激励机制。
